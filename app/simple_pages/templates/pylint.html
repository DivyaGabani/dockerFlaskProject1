Why Pylint?

Pylint is a tool that checks for errors in Python code, tries to enforce a coding standard and looks for code smells. It can also look for certain type errors, it can recommend suggestions about how particular blocks can be refactored and can offer you details about the code's complexity.

It’s highly configurable, having special pragmas to control its errors and warnings from within your code, as well as from an extensive configuration file. It is also possible to write your own plugins for adding your own checks or for extending pylint in one way or another.

Install

Pylint can be simply installed by running

pip install pylint

Upgrading Pylint

pylint --version

To upgrade to the absolute latest version

pip install pylint --upgrade

If you want to install a specific version, run the following command with the version you wish to install.

pip install --force-reinstall pylint==2.7.1


Pylint checks for many things. Some of them may be more important than others for your application. You can create a file that allows you to tell Pylint to ignore certain checks. This file is called .pylintrc.

Run

Once we have installed pylint, we can easily use it by running the command pylint with the name of the file as follows:

pylint filename.py
